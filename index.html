📜 改版歷程細節
🚀 V8.0 基地遷移＋本地自動化架構（2026-05-08）：
  • 系統架構重大變革：捨棄 GitHub Actions 線上運算，改採「本地無菌主算 + 雲端靜態展示」雙軌架構，本地端負責全部運算與 push，GitHub Pages 僅作靜態展示節點。
  • 極速偵測技術：整合 FinMind Cache 快取機制，掃描全市場 API 消耗較舊架構降低 80%，單次完整掃描已可在 1 分鐘內完成。
  • 環境免疫設計（Path Immunity）：實現動態路徑自動定位，支援跨硬碟基地遷移，路徑變動後程式不崩潰、不需手動修改設定。
  • 通訊模組預備：底層程式碼重構完成 Line Notify 擴充介面，後續可一鍵接入訊號推播通知。
🆕 V8.0 互動排序 + 搜尋增強（2026-05-07）：
  • 各魚池排序選擇器：在每個魚池標題列右側新增下拉菜單，支援四種排序模式：預設（法人買超）、法人買超、現價高低、成交量大小，一鍵重排卡片。
  • 模糊搜尋 + 股名支援：搜尋框從精確匹配改為 includes() 模糊搜尋，同時支援股號代碼與股名搜尋，輸入一個字元即可觸發。
  • 模擬倉折疊化：將 Beta user 模擬倉改為 <details> 可折疊元件，預設收合，點擊標題展開，減少首屏視覺噪音。
✅ V7.9 維運日誌 + 自動推送（2026-05-02）：
  • 維運日誌 log_report.json：每次掃描結束後自動產出，記錄 API 消耗次數、快取命中數、處理檔數、執行狀態，供前端儀表板即時讀取。
  • git_sync.py 自動推送：本地執行時掃描完畢自動呼叫 git_sync.py，精準 add 三個戰報（plum_blossom_data.json、ocean_history.json、log_report.json）→ commit → push，無需手動操作。
  • 例假日智慧跳出：偵測非交易日自動跳出，log_report.json 記錄 status: "Skipped-Holiday"，不消耗 API 額度。
✅ V7.8 持久化快取 30h TTL（2026-05-01）：
  • 即時寫盤：每次 FinMind 呼叫成功後立即寫入 finmind_cache.json，程式中斷不會遺失快取。
  • TTL 調整 24h → 30h：確保昨日盤後快取在今日仍可命中，跨日測試不重複消耗 API。
  • 自動清理：啟動時自動清除逾期快取項目，避免快取無限膨脹。
✅ V7.7 FinMind 重試封裝（2026-05-01）：
  • fetch_finmind 重試邏輯封裝：統一封裝 API 呼叫與重試機制，消除 Token 重複消耗問題，失敗時最多重試 2 次再回傳空值。
📊 V7.6 Dashboard 2.0版（2026-05-01）：
  • 視覺化儀表板 Dashboard 2.0：新增三大即時圖表區塊，資料來源為 dashboard_stats 欄位（由 radar.py 每日產出）。
  • ① 魚池多空佔比（Doughnut）：統計全體命名魚池的「買入加碼 vs 靜候觀察」比例，快速掌握整體市場多空情緒。
  • ② 汪洋大魚產業熱力 TOP10（Horizontal Bar）：顯示當日全市場掃描結果中強勢股最集中的前 10 大產業，一眼看出主流資金板塊。
  • ③ 各魚池健康指數（CSS Progress Bar）：每個命名魚池的買入加碼佔比（%），搭配漸層色系呈現強弱分佈。
🚀 V7.5 API 三劍客降載版（2026-05-01）：
  • ① Yahoo MA5 前置預篩：在呼叫 FinMind 籌碼 API 之前，先以 Yahoo 已下載的資料判斷 收盤 >= MA5。若不符合，action 邏輯必為「靜候觀察」，直接跳過 FinMind 呼叫。本次實測攔截 65 支冗餘請求。
  • ② Yahoo 股價取代 FinMind 股價：市場掃描與魚池監控全面改用 Yahoo 批量下載的股價資料計算，不再個別呼叫 FinMind 股價 API（每支節省 1 次）。Yahoo 未命中時才啟動備援，備援失敗才退回 FinMind 最後防線，確保資料完整性。
  • ③ TaiwanStockInfo 7 日快取：股票名稱、產業、市場別等基本資料極少變動，改為獨立 finmind_info_cache.json 儲存，7 天內直接讀本地，不重複消耗 API。
  • 🔐 安全修復：FinMind Token 重新修正為從 GitHub Secrets 環境變數讀取，排除硬碼風險。
  • 📊 實測效果：FinMind API 消耗由 V7.4 的 ~430 次 驟降至 171 次，實測降幅 -60%（超越預估目標）。
✅ V7.4 快取降載與戰術縮圈（2026-04-28）：
  • API 防呆快取：導入本地 finmind_cache.json 機制，同日內重複測試執行不再重複消耗 API 額度。
  • 戰術性縮圈：第一階段粗篩成交量門檻由 1,000 張提升至 1,500 張，過濾冷門股並大幅降低 API 負載。
