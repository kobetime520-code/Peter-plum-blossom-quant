## 💡 隔離的核心價值

### 問題背景
傳統協作中，決策者常被繁雜的程式碼細節淹沒，影響戰略思考質量。

### 解決方案
- ✅ 決策層專注於「做對的決定」與「優化管理邏輯」
- ✅ 執行層專注於「高效產出」與「品質交付」
- ✅ 確保每項指令都經過深度思考且具備完整脈絡

### 預期效果
- 提升決策質量 ↑
- 降低執行風險 ↓
- 加速團隊協作 ↑
- 減少溝通成本 ↓

## 🔥 最高指導原則

> **不斷為 Claude 準備最好、最精準的子彈（Prompt）**

每一道交付給 Claude 的指令都應滿足以下標準：

### 精確清晰 🎯
- 明確的任務目標
- 清楚的輸出標準
- 無歧義的需求描述

### 脈絡完整 📋
- 必要的背景資訊
- 決策邏輯說明
- 約束條件列舉

### 經過驗證 ✓
- 已由決策層深思熟慮
- 與組織策略對齊
- 資源與時間評估完成

## 📋 各角色執行檢查清單

### 決策層（Gemini、Sidey、各 SKILL 持有人）

**✓ 制定指令前**
- [ ] 明確任務目標與輸出標準
- [ ] 梳理完整的業務脈絡
- [ ] 評估資源與時間需求
- [ ] 與上游決策對齊

**✓ 交付指令時**
- [ ] 提供完整背景資訊
- [ ] 列舉約束條件與邊界
- [ ] 說明戰略意圖
- [ ] 確認執行路徑清晰

**✓ 執行完成後**
- [ ] 驗證輸出品質
- [ ] 評估指令有效性
- [ ] 記錄改進方案
- [ ] 優化下一輪指令

### 執行層（Claude、各技術人員）

**✓ 接收指令時**
- [ ] 確認理解指令意圖
- [ ] 確認約束條件清晰
- [ ] 確認資源與時間充足

**✓ 執行過程中**
- [ ] 按指令標準產出
- [ ] 及時反饋異常情況
- [ ] 提出執行建議

**✓ 交付成果時**
- [ ] 驗證符合標準
- [ ] 提供完整文檔
- [ ] 總結執行心得

## 📞 Q&A

**Q：這個決策何時生效？**  
A：即刻生效。請各單位立即調整工作流程。

**Q：如何評估指令品質？**  
A：指令應包含明確的目標、完整脈絡、清楚的約束條件。如有疑問，決策層應與執行層確認。

**Q：執行反饋如何提交？**  
A：各事業群每週向 Sidey 提交執行報告，包含「完成度」「品質評分」「改進建議」。

**Q：現有進行中的任務如何調整？**  
A：逐步調整。優先完成現有任務，新任務按新流程執行。
