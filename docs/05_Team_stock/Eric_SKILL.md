## 引擎已內建的籌碼欄位（V8.7 實況）

`calculate_stock_data` 已將 FinMind 法人資料拆解計算，Eric 負責解讀以下系統欄位：

| 欄位 | 計算 | 說明 |
|---|---|---|
| `inst_buy` | 三大法人 net_buy 加總（張） | 30 日窗口（實際抓 45 日曆日範圍加總） |
| `foreign_buy` | 外資/外陸資 net_buy（張） | 已從合計中**獨立拆分** |
| `trust_buy` | 投信 net_buy（張） | 已從合計中**獨立拆分** |
| `chip_signal` | 標籤 | 雙買 / 投信單買 / 外資單買 / 無買 |
| `inst_grade` | 分級 | S / A / B / C / X |

**chip_signal 判斷**：外資>0 且 投信>0 → 雙買；僅投信>0 → 投信單買；僅外資>0 → 外資單買；皆否 → 無買

**inst_grade 分級**：≥5,000 張 → S；≥1,000 → A；≥500 → B；>0 → C；其餘 → X

### strength_score 籌碼評分 `_calc_chip_score`（上限 35，Eric 主責）

- 法人方向：外資+投信雙買 +15；僅投信 +10；僅外資 +7
- 淨買張數：≥5,000 +20；≥1,000 +15；≥500 +10；>0 +5

> 動作訊號連動：`inst_buy_30d > 0` 是 action 進入「買入加碼」的籌碼門檻（搭配 close ≥ MA5）。

### V9.0 A1 籌碼方向閘門（2026-06-08，Eric 主責解讀）

汪洋大魚入池的籌碼條件自 V9.0 起升級為**雙重驗證**：

```
舊制（≤V8.9）：inst_buy（合計淨買超）> 0 即通過
新制（V9.0）：inst_buy > 0 且（foreign_buy > 0 或 trust_buy > 0）
```

- **目的**：剔除「假合計正」——外資與投信雙雙賣超、僅靠自營商買超撐正合計數的個股，籌碼方向實質偏空
- **攔截統計**：被 A1 擋下的檔數寫入戰報 `yf_skipped_chip`
- **Eric 解讀職責**：
  - `yf_skipped_chip` 突然放大 → 市場出現大量「自營商獨撐」個股，短線波動風險升高，主動回報 JW
  - 固定魚池個股若僅自營商買超（外資投信皆賣），比照 A1 邏輯標記 🔴 籌碼轉弱，即使 inst_buy 為正
